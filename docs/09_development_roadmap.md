# forin — 개발 로드맵

**버전**: 1.1.0 | **작성일**: 2026-04-17 | **변경**: 백엔드 NestJS → Go/Gin

---

## MVP 범위 (v0.1)

**목표**: 10주 내 TestFlight/Play Console 내부 베타 출시  
**핵심 조건**: 간호사 1명이 처음부터 끝까지 의미 있는 학습 경험을 완성할 수 있어야 한다.

### MVP 포함 기능
| 기능 | 상세 |
|------|------|
| 인증 | 이메일 + Google OAuth |
| 온보딩 | 5단계 전체 + 튜토리얼 스테이지 |
| 직업 | 간호사만 |
| 국가 트랙 | 호주(AU)만 |
| 커리큘럼 | Module 1 + Module 2 (약 20개 스테이지) |
| 문제 유형 | 4가지 전부 |
| AI 채점 | 대화 연습 LLM 채점 + 키워드 폴백 |
| 목숨 시스템 | 5하트, 30분 자동 충전 |
| XP / 레벨 | 10레벨 시스템 |
| 별점 | 3별 시스템 |
| 스트릭 | 일일 스트릭 추적 (방어막 제외) |
| 고양이 | 모자/의상/배경 3슬롯 + 아이템 20개 |
| 선물 상자 | Basic/Silver 2종 |
| 업적 | 5개 기본 업적 |
| 알림 | 일일 리마인더 + 스트릭 경고 2종 |

### MVP 제외 기능 (Phase 2+)
| 기능 | 이유 |
|------|------|
| 의사/약사 직업 | 커리큘럼 제작 시간 |
| 영국/캐나다 트랙 | 콘텐츠 현지화 필요 |
| 음성 입력 | 추가 AI 비용, MVP 후 우선순위 결정 |
| 소셜 기능/리더보드 | MVP 핵심 가치와 무관 |
| 프리미엄 구독/결제 | 법적 검토, 앱스토어 설정 필요 |
| 스트릭 방어막 | 간단하지만 MVP 후 추가 가능 |
| Cat Shop | 인벤토리만 MVP에 포함 |

## 개발 일정 (10주)

### Phase 0: Foundation (1-2주)

**목표**: 개발 환경 완성, 기본 인증 동작

**백엔드 작업** (Go/Gin):
- [ ] Go 프로젝트 초기화 (`go mod init`, 디렉토리 구조 세팅)
- [ ] Docker Compose 설정 (PostgreSQL 16, Redis)
- [ ] GORM + golang-migrate 연결, 전체 DB 스키마 마이그레이션 실행
- [ ] Gin 서버 기본 설정 (미들웨어: 로깅, CORS, Rate Limiting)
- [ ] Auth 핸들러/서비스 (이메일 회원가입/로그인, JWT, Google OAuth)
- [ ] 시드 데이터: professions, 기본 커리큘럼 1모듈, 3스테이지, 15문제
- [ ] swaggo/gin-swagger로 API 문서 자동 생성
- [ ] Makefile (build, run, migrate, seed 명령어)

**모바일 작업**:
- [ ] React Native Expo 프로젝트 초기화
- [ ] 네비게이션 구조 설정 (Bottom Tab + Stack)
- [ ] 디자인 시스템: 컬러 팔레트, 타이포그래피, 기본 컴포넌트 (Button, Card, ProgressBar)
- [ ] API 클라이언트 설정 (TanStack Query, Axios 인터셉터)
- [ ] 인증 화면 (회원가입/로그인/Google OAuth)

**완료 기준**: 에뮬레이터에서 회원가입 → 로그인 동작 확인

---

### Phase 1: Core Learning Loop (3-5주)

**목표**: 문제 풀기 → 결과 확인 전체 흐름 동작

**백엔드 작업** (Go):
- [ ] Curriculum API (모듈/유닛/스테이지/문제 조회)
- [ ] 4가지 문제 유형별 채점 서비스
  - `internal/service/evaluator/sentence_arrangement.go` (정규화 후 완전 일치)
  - `internal/service/evaluator/word_puzzle.go` (빈칸별 독립 판정)
  - `internal/service/evaluator/meaning_match.go` (완료 시간/오매칭 수 기반)
  - `internal/service/evaluator/conversation.go` (Claude API + goroutine 비동기, 키워드 폴백)
- [ ] Stage Attempt CRUD (시작/문제제출/완료)
- [ ] XP 지급 및 레벨 계산 서비스
- [ ] 목숨 차감 및 자동 충전 로직 (Redis TTL 활용)

**모바일 작업**:
- [ ] 홈 화면 (커리큘럼 카드, 진도 바, 목숨 표시)
- [ ] 스테이지 소개 화면
- [ ] 4가지 문제 유형 UI
  - 문장 배열: 드래그앤드롭 타일
  - 단어 퍼즐: 빈칸 탭 선택
  - 의미 매칭: 뒤집기 카드 그리드
  - 대화 연습: 텍스트 입력 + AI 피드백 표시
- [ ] 스테이지 완료 화면 (별점, XP, 요약)
- [ ] 정답/오답 애니메이션 피드백

**완료 기준**: 스테이지 1개를 처음부터 끝까지 플레이하고 XP를 받을 수 있음

---

### Phase 2: Onboarding + Gamification (6-7주)

**목표**: 온보딩 완성, 고양이 + 선물 상자 시스템 동작

**백엔드 작업** (Go):
- [ ] 레벨 평가 퀴즈 API (10문항 + 레벨 배정 로직)
- [ ] 선물 상자 드롭율 알고리즘 `internal/service/gift_box_service.go` (가중치 랜덤)
- [ ] 인벤토리 관리 서비스 (중복 감지 → Catnip 전환)
- [ ] 고양이 아이템 장착 API
- [ ] 업적 달성 감지 + 보상 지급 서비스

**모바일 작업**:
- [ ] 온보딩 화면 5단계 전체
- [ ] 레벨 평가 퀴즈 UI
- [ ] 고양이 캐릭터 렌더링 (레이어드 이미지 합성)
- [ ] 선물 상자 개봉 애니메이션 (Lottie)
- [ ] 인벤토리 화면
- [ ] 고양이 커스터마이징 화면 (슬롯 탭 + 실시간 프리뷰)

**콘텐츠 작업**:
- [ ] 고양이 아이템 20개 일러스트 완성 (3슬롯 × 희귀도별)
- [ ] Basic/Silver 상자 개봉 Lottie 애니메이션

**완료 기준**: 온보딩 → 첫 스테이지 → 상자 개봉 → 고양이 장착 전체 플로우 동작

### Phase 3: Progress + Polish (8-9주)

**목표**: 진행도 시스템 완성, 알림 연동, 콘텐츠 20스테이지 완성

**백엔드 작업** (Go):
- [ ] 스트릭 추적 (일일 활동 감지, 스트릭 업데이트)
- [ ] 모듈/유닛 잠금 해제 자동 계산
- [ ] 푸시 알림 서비스 (Expo Push API)
  - Go `time.AfterFunc` / cron job으로 일일 리마인더 스케줄러
  - 스트릭 경고 (22시간 무활동 Redis 키 만료 감지)
- [ ] 주간 학습 통계 API

**모바일 작업**:
- [ ] 커리큘럼 맵 화면 (모듈/유닛/스테이지 잠금/해제 상태)
- [ ] 업적 화면
- [ ] 레벨업 축하 애니메이션 (Lottie)
- [ ] 스트릭 위험 배너
- [ ] 알림 설정 화면
- [ ] 프로필/통계 화면

**콘텐츠 작업**:
- [ ] 스테이지 콘텐츠 20개 전체 완성 (RN 컨설턴트 검수 완료)
- [ ] 오디오 녹음 완료 (~160트랙)

**완료 기준**: 20개 스테이지 전부 플레이 가능, 스트릭 + 알림 동작

### Phase 4: QA + Launch Prep (10주)

**목표**: 베타 테스트 완료, 앱스토어 제출 준비

- [ ] 내부 QA: 4가지 문제 유형 전체 엣지 케이스 테스트
- [ ] 성능 프로파일링

- [ ] Sentry 에러 모니터링 연동 (Go SDK + React Native SDK)
- [ ] Mixpanel 분석 이벤트 연동
  - `session_start`, `stage_complete`, `gift_box_open`, `level_up`, `onboarding_complete`
- [ ] TestFlight (iOS) + Play Console 내부 트랙 배포
- [ ] 타겟 사용자 10-20명 베타 테스트 (현직 간호사 대상)

## 기술 리스크 및 완화 방안

| 리스크 | 가능성 | 영향 | 완화 방안 |
|--------|--------|------|---------|
| AI 채점 레이턴시 (>3초) | 높음 | 높음 | goroutine 비동기 처리, 유사 응답 Redis 캐싱, 폴백(키워드 방식) 즉시 전환 |
| iOS 앱스토어 심사 거절 | 중간 | 높음 | Apple HIG 완전 준수, TestFlight 선행 테스트 |
| 오디오 CDN 비용 규모화 | 중간 | 중간 | 64kbps 압축, CDN 캐싱 헤더 최대화, 지연 로딩 |
| 콘텐츠 임상 정확도 오류 | 중간 | 매우 높음 | 전담 RN 컨설턴트 주1회 리뷰, OTA 즉시 수정 |
| 대화 평가 프롬프트 인젝션 | 낮음 | 높음 | 사용자 입력 → 시스템 프롬프트 완전 분리, Go에서 입력 sanitize |
| PostgreSQL 진도 쿼리 성능 | 중간 | 중간 | 복합 인덱스 (user_id + stage_id), 홈 화면 Redis 캐싱 5분 |
| Go 단일 서버 병목 | 낮음 (초기) | 중간 | Go 고루틴 특성상 단일 서버도 높은 동시성 처리 가능, 필요 시 수평 확장 |
