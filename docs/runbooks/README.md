# Runbooks — 개발/운영 트러블슈팅

개발 및 운영 중 발생한 이슈의 진단 절차와 재발 방지 가이드를 모아둔 문서 모음입니다.

각 런북은 다음 구조를 따릅니다:

1. **증상** — 사용자/개발자가 목격한 현상
2. **근본 원인** — 실제로 무엇이 잘못되었는지
3. **진단 절차** — 동일 증상을 재현/확인하는 명령
4. **해결** — 즉시 복구 단계
5. **재발 방지** — 체크리스트, 관련 코드 경로
